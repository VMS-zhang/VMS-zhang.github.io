RAII 和 智能指针

Posted on 2022-05-02 Edited on 2022-05-03 In cpp

RAII

RAII（Resource Acquisition Is Initialization）是由c++之父Bjarne Stroustrup提出的，中文翻译为资源获取即初始化，他说：使用局部对象来管理资源的技术称为资源获取即初始化；这里的资源主要是指操作系统中有限的东西如内存、网络套接字等等，局部对象是指存储在栈的对象，它的生命周期是由操作系统来管理的，无需人工介入；

原理

资源使用一般是三个步骤：

  1. 获取
  2. 使用
  3. 销毁

如果忘记了销毁，会造成资源泄漏，导致程序崩溃。 为了让资源自动的销毁，Bjarne给出的解决方案是：RAII。

其原理是 C++ 的局部变量会在退出其作用域的时候自动的被销毁；根据这个特性，可以让局部变量被销毁的时候把资源进行释放。

总结下来，RAII 的主要步骤可以分为四步：

  1. 设计一个封装资源的类
  2. 在构造函数中进行资源申请
  3. 使用资源的时候，通过声明一个该 RAII 类的对象来使用资源
  4. 在析构函数中释放资源

使用 new 和 delete 来进行动态内存的管理是很困难的，无法保证正确的释放。
为了更安全、高效的使用动态内存，c++11按照 RAII 的思想，提供了两种智能指针：shared_ptr 和 unique_ptr。

  • shared_ptr:允许多个指针指向同一个对象
  • unique_ptr:独占所指向的对象

另外提供了 wwak_ptr 的伴随类，其是一个弱引用，指向 shared_ptr所管理的对象。

三者都在<memory>头文件中。

智能指针也是模板类型，所以定义智能指针的时候必须提供其可以指向的类型，在尖括号内给出定义。

引用计数

每个 shared_ptr内部都有一个计数器，即引用计数。无论何时拷贝一个shared_ptr，计数器都会递增。

计数器递增的场景：

  1. 用一个shared_ptr初始化另一个shared_ptr的时候
  2. 将shared_ptr作为一个参数传递给函数的时候
  3. shared_ptr作为函数返回值的时候

一旦一个shared_ptr的计数器变为零，他会自动释放自己所管理的对象。

使用方式

智能指针的使用方式： 

shared_ptr和 unique_ptr 共有操作 描述
shared_ptr sp 空智能指针，可以指向类型为 T 的对象
unique_ptr up 空智能指针，可以指向类型为 T 的对象
p 将 p 作为一个条件判断，若 p 指向一个对象，则为 True
*p 解引用p,获取其指向的对象
p.get() 返回 p 中保存的指针。**小心使用，若智能指针释放了其对象，返回的指针所指向的对象也就消失了
swap(p,q) 交换 p，q 的指针
shared_ptr独有操作 描述
make_shared(args) 返回一个 shared_ptr，指向一个动态分配的类型为 T 的对象，使用 args 初始化此对象
shared_ptrp(q) p 是 shared_ptr的拷贝，会传递 q 中的计数器。q 中的指针必须可以转换成 T*
p=q 会递减 p 的引用计数，递增 q 的引用计数；如果 p 的引用计数变为0，则释放其原内存
p.unique() 如果 p.use_count 为1返回 true，否则返回 false
p.use_count() 返回于 p 共享对象的智能指针的数量，性能较低，主要用于调试

  2. 如果不指定任何参数，则进行的是值初始化。

unique_ptr

unique_ptr与 shared_ptr的主要区别是：某个时刻只能有一个 unique_ptr指向一个给定对象。
unique_ptr被销毁的时候，其对应的对象也被销毁。

weak_ptr

weak_ptr 是一种不控制所指向对象生存期的智能指针，他指向一个有 shared_ptr管理的对象。
把一个 weak_ptr绑定到 shared_ptr不会改变shared_ptr的引用计数。

  • 在 shared_ptr的最后一个对象被销毁的时候，其对应的内存对象也会被销毁，此时即使 weak_ptr指向这个对象，对象也还是会被释放！
