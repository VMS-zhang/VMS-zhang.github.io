隐式转换

c++ 中，某些类型之间是有关联的。如果两个类型有关联，那么在使用一种类型的对象时候，可以使用另一种类型的对象 or 值来替代。

如果两个类型可以相互转换，那么他们就是关联的。

2.1是 double 类型，2是 int 类型，因为类型不同，所以不能直接相加，需要根据类型转换规则来讲运算对象的类型进行统一后再求值，这个类型转换是自动进行的，即隐式转换。

算数类型之间的隐士转换一般尽可能的避免精度损失。大部分时候，如果既有浮点数又有整型数，一般是把整型数替换为浮点型。所以刚才的例子2被转换成 double之后再和2.1求值，所得的结果是 double。

因为 a 的类型已经被确定为 int 了，所以需要把2.1+2得到的 double 值转换为 int 类型的值。转换过程中忽略了小数部分，所以结果是4。

隐式转换时机

  1. 大部分情况下，比 int 类型小的整型值提升为较大的类型。
  2. 在条件表达式中，非布尔转换为布尔。
  3. 初始化的时候，初始化的值转换为变量的类型（上文程序的情况）；赋值时候，右侧类型转换为左侧类型。
  4. 算术运算 or 关系运算如果有多种类型，需要转换成同一种类型。
  5. 函数调用时候可能也会有类型转换。

显示转换

强制将对象转换成另外一种类型。
如果想执行浮点数除法，则需要将 i 和 j 显式转换为 double 类型。

有时不得不进行强制类型转换，但是强制类型转换是非常危险的！！

c++11中提供了四种类型转换，分别是 static_cast、const_cast、reinterpret_cast、dynamic_cast;

static_cast

只要底层不包含 const，都可以执行 const_cast。

  1. 在把大精度的值转换成小精度的值的时候，static_cast很有用（如果不在乎精度损失的话）。
  1. 也可以用在编译器无法自动执行的类型转换上，例如找回 void* 的数据。

但是注意，如果强制转换的结果和原始的类型不同，那会产生未定义的结果。

const_cast

const_cast的唯一作用是改变对象底层 const。
可以去掉const，也可以加上 const。但是去掉 const 之后的写行为是未定义的。

const_cast的作用只是改变 const 属性，不能改变表达式的类型！

reinterpret_cast

reinterpret_cast的作用是为运算对象的位模式提供较低层次上的重新解释。
行为是很危险的，尽量不要用。

dynamic_cast

主要作用是在继承关系上进行转换。
Base 至少有一个虚函数。

explicit

对于一个类来说，如果其（有一个或以上的）构造函数只接受一个实参，那么这个构造函数实际上定义了从实参类型转换为此类类型的隐士转换机制。

类 A 有两个只有一个实参的构造函数，那么我们可以将实参的类型转换为类 A 的类型。

只允许一步转换！
在string 转换为类 A 的时候，不能使用 A b= “123”这样的形式，因为编译器只会自动执行一步类型转换。A b= “123”实际使用了两种转换规则，先将”123”转换为 string，再将 string 转换为 A 类型。可以使用上文的形式也可以使用：A c = string(“456”);

如果程序不想要对某个类类型进行自动的隐式转换，那么可以把构造函数中声明为 explicit。

注意 explicit 只对一个实参的构造函数有效！

new 和 delete

c++使用 new 和 delete 关键字来进行分配和释放动态内存。通过 new 申请内存之后要通过 delete 释放掉，否则会产生内存泄露。

初始化

new 初始化一块内存之后，使用的是默认初始化，则：

  • 内置类型和组合类型的对象的值是未定义的
  • 类类型对象使用默认构造函数进行初始化

new 时候如果可以指导初值，可以使用圆括号 or 列表初始化 or 值初始化。

auto

new 可以配合 auto 进行使用：

  1. 可以根据new 中指定的类型进行推断
  2. 可以根据 new 中参数的类型进行推断
  3. 只有括号中仅有一个单一初始化器时候才可以进行推断

delete

如果每次 new 之后不进行 delete，则程序一直运行下去总有一天内存会耗尽（因为内存泄露）。
所以需要使用 delete 来对 new 申请的内存进行释放。

使用 delete 的时候，需要注意以下几点：

  1. 不要释放非 new 分配的内存，其行为是未定义的
  2. 不要重复释放内存，行为是未定义的
  3. 传给 delete 的必须是指针 or 空指针

不用使用已经释放的对象 ：在释放之后把指针置nullptr，有时可以检测这种错误

在 delete 之后，指针值就无效了。指针虽然已经失效，但是其依旧保存着之前指向的地址的值，指针也就变成了空悬指针。

生命周期

new 出来的对象的生命周期指导 delete 为止。如果 new 之后没有调用 delete，则 new 出来的对象一直存在！！

cppreference.com说明了自动构造initializer_list的场景:

A std::initializer_list object is automatically constructed when:

  1. a braced-init-list is used to list-initialize an object, where the corresponding constructor accepts an std::initializer_list parameter
  2. a braced-init-list is used as the right operand of assignment or as a function call argument, and the corresponding assignment operator/function accepts an std::initializer_list parameter
  3. a braced-init-list is bound to auto, including in a ranged for loop

翻译过来大致如下：

  1. 当初始化的时候使用的是大括号初始化，被自动构造。包括函数调用时和赋值。例如 vector<int> arr{1,2,3,4} 使用的就是 initializer_list。
  2. 当涉及到for(initializer: list) ,list被自动构造成initializer_list对象。
