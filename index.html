c++自动类型推断

Posted on 2022-05-02 Edited on 2022-05-03 In cpp

c++11 提供了两种自动类型推断的方式：

  1. auto
  2. decltype

auto

auto让编译器去分析表达式的类型，与原来的特定的类型说明符（例如 double）不同的是，auto 通过让编译器使用初始值来推算变量的类型。所以：auto定义的变量必须有初始值。

  • 如果 a 和 b 是 int 的对象，那 c 的类型就是 int
  • 如果 a 和 b 是 class A 的对象，那么 c 的类型就是 class A

  1. auto 可以一次性生命多个变量，但是多个变量的类型必须相同。

    自动推断出了 c 的类型是 A，d 的类型是 A*。

  2. 编译器推断出来的 auto 类型有时候可能和初始值的类型不一样

  • 在使用引用的时候实际使用的是引用的对象，特别是当引用被用作初始值的时候，真正参与初始化的其实是引用对象的值。
    此时编译器自动以引用对象的类型作为 auto 的类型：

decltype

decltpye:想要使用表达式的类型推断出来定义的变量的类型，而又不想使用该表达式的类型的值来初始化变量；其主要作用是返回操作数的数据类型，在此过程中，编译器分析表达式并得到他的类型，但不实际计算其值。

1.如果 decltype 使用的表达式是一个变量，那么 decltype 返回该变量的类型（包括 const 和引用）

2.如果 decltype 使用的表达式不是一个变量，那么返回表达式结果对应的类型

  • 如果表达式的求值结果是左值，那么 decltpye作用于该表达式（不是变量）得到一个引用类型

3.对于 decltype 所用的表达式，如果变量名加上()，得到的结果和不加()不同

  • 如果加上一个 or 多个（），编译器会把变量看做表达式
  • 不加括号，则得到变量的类型

decltpye((variable)) 双层括号的结果永远是引用;decltpye(variable)只有当变量本身是引用类型时候才是引用。

RAII 和 智能指针

Posted on 2022-05-02 Edited on 2022-05-03 In cpp

RAII

RAII（Resource Acquisition Is Initialization）是由c++之父Bjarne Stroustrup提出的，中文翻译为资源获取即初始化，他说：使用局部对象来管理资源的技术称为资源获取即初始化；这里的资源主要是指操作系统中有限的东西如内存、网络套接字等等，局部对象是指存储在栈的对象，它的生命周期是由操作系统来管理的，无需人工介入；

原理

资源使用一般是三个步骤：

  1. 获取
  2. 使用
  3. 销毁

如果忘记了销毁，会造成资源泄漏，导致程序崩溃。 为了让资源自动的销毁，Bjarne给出的解决方案是：RAII。

其原理是 C++ 的局部变量会在退出其作用域的时候自动的被销毁；根据这个特性，可以让局部变量被销毁的时候把资源进行释放。

总结下来，RAII 的主要步骤可以分为四步：

  1. 设计一个封装资源的类
  2. 在构造函数中进行资源申请
  3. 使用资源的时候，通过声明一个该 RAII 类的对象来使用资源
  4. 在析构函数中释放资源

使用 new 和 delete 来进行动态内存的管理是很困难的，无法保证正确的释放。
为了更安全、高效的使用动态内存，c++11按照 RAII 的思想，提供了两种智能指针：shared_ptr 和 unique_ptr。

  • shared_ptr:允许多个指针指向同一个对象
  • unique_ptr:独占所指向的对象

另外提供了 wwak_ptr 的伴随类，其是一个弱引用，指向 shared_ptr所管理的对象。

三者都在<memory>头文件中。

智能指针也是模板类型，所以定义智能指针的时候必须提供其可以指向的类型，在尖括号内给出定义。

隐式转换

c++ 中，某些类型之间是有关联的。如果两个类型有关联，那么在使用一种类型的对象时候，可以使用另一种类型的对象 or 值来替代。

如果两个类型可以相互转换，那么他们就是关联的。

2.1是 double 类型，2是 int 类型，因为类型不同，所以不能直接相加，需要根据类型转换规则来讲运算对象的类型进行统一后再求值，这个类型转换是自动进行的，即隐式转换。

算数类型之间的隐士转换一般尽可能的避免精度损失。大部分时候，如果既有浮点数又有整型数，一般是把整型数替换为浮点型。所以刚才的例子2被转换成 double之后再和2.1求值，所得的结果是 double。

因为 a 的类型已经被确定为 int 了，所以需要把2.1+2得到的 double 值转换为 int 类型的值。转换过程中忽略了小数部分，所以结果是4。

隐式转换时机

  1. 大部分情况下，比 int 类型小的整型值提升为较大的类型。
  2. 在条件表达式中，非布尔转换为布尔。
  3. 初始化的时候，初始化的值转换为变量的类型（上文程序的情况）；赋值时候，右侧类型转换为左侧类型。
  4. 算术运算 or 关系运算如果有多种类型，需要转换成同一种类型。
  5. 函数调用时候可能也会有类型转换。

new 和 delete

c++使用 new 和 delete 关键字来进行分配和释放动态内存。通过 new 申请内存之后要通过 delete 释放掉，否则会产生内存泄露。

初始化

new 初始化一块内存之后，使用的是默认初始化，则：

  • 内置类型和组合类型的对象的值是未定义的
  • 类类型对象使用默认构造函数进行初始化

new 时候如果可以指导初值，可以使用圆括号 or 列表初始化 or 值初始化。

auto

new 可以配合 auto 进行使用：

  1. 可以根据new 中指定的类型进行推断
  2. 可以根据 new 中参数的类型进行推断
  3. 只有括号中仅有一个单一初始化器时候才可以进行推断

delete

如果每次 new 之后不进行 delete，则程序一直运行下去总有一天内存会耗尽（因为内存泄露）。
所以需要使用 delete 来对 new 申请的内存进行释放。

使用 delete 的时候，需要注意以下几点：

  1. 不要释放非 new 分配的内存，其行为是未定义的
  2. 不要重复释放内存，行为是未定义的
  3. 传给 delete 的必须是指针 or 空指针

不用使用已经释放的对象 ：在释放之后把指针置nullptr，有时可以检测这种错误

在 delete 之后，指针值就无效了。指针虽然已经失效，但是其依旧保存着之前指向的地址的值，指针也就变成了空悬指针。

生命周期

new 出来的对象的生命周期指导 delete 为止。如果 new 之后没有调用 delete，则 new 出来的对象一直存在！！
